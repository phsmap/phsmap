As a result, the school has requested that I implement an access control that only grants website access to MCPS students/staff + authorized visitors.

  • This also means that should you decide to make any "export" of this application's data (i.e. by taking screenshots or printing the map out), you may not share it with anyone except for other Poolesville students, staff and temporary authorized visitors (such as students visiting our school for an event, parents, and subs).



    • (3) Use of Account Information + Data Security Notice

    Use of Account Information

  • The resource server (which is responsible for serving map/layer data to users) controls access by looking up incoming users' Microsoft profiles and checking to make sure that they are of MCPS origin.

  • During this process, the server is ABLE TO SEE a limited amount of account data, such as email address, phone number, office location, job title and language preference. However, most of it is not recorded since it isn't actually used to authenticate users. (For more information about what information is visible when you log in with Microsoft see, this article).

  • The server will RECORD IP addresses, user names, user agents, email addresses, date/time of first and subsequent sign ins, and date/time of requests for protected resources.

    • Data Security

  • I would like to restate that this app DOES NOT HANDLE any user passwords or secrets, and that it CANNOT ACCESS OR ACT ON BEHALF OF YOUR ACCOUNT.

  • Users enter their passwords on Microsoft's website, after which Microsoft gives us an access token, which is essentially a digitally signed, temporary certificate that can be used to prove a user's identity and allow the bearer (in this case, the PHS Interactive Map) limited access to account information.

  • This is different from some other applications, where your username/password is saved by the application and then later used to act directly on your behalf -- which is bad because a password carries full account access priviledges, does not expire and can be changed to lock the original account owner out.

  • For more information on how this system works and how it prevents 3rd party applications such as this one from accessing your password and your account, see this document about OAuth2.0.
